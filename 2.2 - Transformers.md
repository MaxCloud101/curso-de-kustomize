# Transformers

Los Transformers son mecanismos que modifican recursos YAML automáticamente.

```
YAML original
    ↓
Transformers
    ↓
YAML transformado
```

Los transformers NO crean recursos nuevos.

Lo que hacen es:

- modificar
- agregar
- reemplazar
- transformar

partes del YAML.

## Transformers

- namePrefix: Agrega prefijo (ej: project2.2.1)
- nameSuffix: Agrega sufijo (ej: project2.2.1)
- namespace: Cambia namespace (ej: project2.2.1)
- labels: Agrega labels (ej: project2.2.1)
- commonAnnotations: Agrega annotations (ej: project2.2.1)
- images: Cambia imágenes (ej: project2.2.1)
- replicas: Cambia replicas (ej: project2.2.1)

- replacements Transformer: copiar valores de un recurso hacia otro recurso (ej. project2.2.2)
- Transformer Custom: permite crear transformaciones personalizadas sobre YAML. (ej. project2.2.3)

## Transformers Patches

- Strategic Merge Patch:permite modificar partes específicas de un recurso Kubernetes existente usando un YAML parcial. (ej. project2.2.4)
- JSON6902 Patch: modifica recursos usando operaciones explícitas (add, replace, remove, move, copy, test) sobre rutas YAML convertidas a formato JSON Path. (ej. project2.2.5)
